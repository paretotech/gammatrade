Gateway setup (paper account)
  1. 1.
    Download IB Gateway from interactivebrokers.com — pick the stable build for your OS. Lighter than TWS (~150MB).
  2. 2.
    Launch Gateway, log in to your paper account. Pick IB API mode (not FIX) on the login screen. The 2FA push will hit your IBKR Mobile app.
  3. 3.
    Enable the API in Configuration → Settings → API → Settings:
    • Enable ActiveX and Socket Clients ✓
    • Read-Only API — unchecked (we need to place orders)
    • Socket port: 7497 (paper)
    • Trusted IPs: add 127.0.0.1
    • Allow connections from localhost only ✓
  4. 4.
    Restart this server with the right env vars. BROKER_READONLY=1 is the default (broker refuses orders) — keep it ON until you're ready to actually trade. 
    pkill -f "automation.server"
# Paper account (port 7497):
export BROKER=ibkr_paper
# OR live account during setup (port 7496) — read-only stays ON:
export BROKER=ibkr_live
python3 -m automation.server
    Live setup is safe in read-only mode — Gateway won't accept orders even if buggy code tries.
  5. 5.
    Refresh this page. Status flips to "Connected" with your paper account ID. If not, check the Last error section above and Gateway's API logs.
Daily 2FA: IBKR forces fresh login every ~24h. Keep Gateway running between sessions; on 2FA prompt accept the IBKR Mobile push. Server stays up — Gateway reconnect is transparent. Hit Reconnect above if the connection wedges.
